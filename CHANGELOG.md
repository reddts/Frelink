# Changelog

## 2026-03-19

### 稳定性、安全与性能修复
- 修复 API 入口识别错误：统一 `version` 请求头读取逻辑，避免接口请求被误判为前台/移动端入口。
- 优化动态路由与默认首页配置加载：为 `route_rule`、默认首页菜单等高频配置增加缓存，减少每次请求的数据库访问。
- 移除源码中的敏感配置依赖：数据库连接改为从环境变量读取，入口密钥与 IV 改为从 `.env` 加载，避免明文写入仓库。
- 修复入口文件启动时序问题：`public/index.php` 改为在容器初始化前使用原生方式读取 `.env`，避免触发 `app()` 未初始化导致的致命错误。
- 优化搜索性能：收敛 `RegexpSearch` 的关键词处理与总数统计逻辑，避免整条联合查询重复执行。
- 优化问题列表与评论列表查询链路：批量加载投票、关注、举报、回答用户等数据，减少首页、问题页的 N+1 查询。
- 修复话题下拉接口分页总数计算逻辑，避免 `count()` 与分页条件混用带来的结果偏差。
- 将前台 SQL 性能采样日志与自动 sitemap 刷新改为环境开关控制，默认关闭，避免把额外 I/O 挂在普通用户请求链路上。
- 修复 sitemap 主机名规范问题：生成链接时统一规范到 `https://www.frelink.top`，避免裸域或错误 Host 被搜索引擎抓取后判定为 404。
- 修复多处 AJAX/接口型控制器在非 XHR 访问下的响应类型问题：`result()` 统一固定为 JSON，避免爬虫直接访问时把数组按 HTML 输出而触发 `variable type error: array`。
- 修复关注问题用户等接口的显式 JSON 返回，并同步清理后台 `_list` 列表与删除接口中的数组直接返回，减少搜索抓取和直链访问时的 500 风险。
- 完成线上发布与回归验证，修复 `www.frelink.top` 首页因入口配置读取时机不当引发的致命错误，站点已恢复正常访问。

## 2026-03-18

### 前台兼容与样式优化
- 修复 PHP 8.2 下的多处兼容问题，包括话题描述为空、权限匹配空值、分词类动态属性等导致的页面异常。
- 修复空搜索页渲染问题，空关键词访问搜索页时不再出现空白。
- 优化移动端导航：重做折叠菜单、搜索区、登录态操作区和语言切换布局，关闭移动端搜索联想浮层，减少遮挡和错位。
- 优化移动端分页：分页区改为横向滚动，避免页码溢出视口。
- 优化文章页移动端工具栏：桌面端保留原有左侧工具条，移动端改为侧边手柄唤起的紧凑工具组，减少对阅读区域的干扰。
- 统一文章、单页、公告、问题、回答、帮助页的正文排版，补充标题、摘要、正文、代码块、表格、列表、引用等样式规范。
- 优化评论区和操作区按钮样式，统一交互视觉风格。
- 清理旧富文本残留，规范空链接、冗余属性和嵌套代码块结构。
- 优化页脚信息层级，新增隐私声明和用户协议入口，并减轻视觉重量。
- 修复文章页、隐私页、协议页、帮助页等多处移动端间距与内容容器问题。
