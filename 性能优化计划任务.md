# 性能优化计划任务

## 目标
- 缩短首页打开时间（首字节时间 + 首屏渲染时间）
- 降低首页请求的数据库查询次数与阻塞资源体积

## 一、立即执行（高优先级）
- [x] 生产环境关闭调试与SQL监听
  - `.env`：`APP_DEBUG=false`、`DATABASE.DEBUG=false`
  - `config/database.php`：`trigger_sql=false`
- [x] 开启字段缓存
  - `config/database.php`：`fields_cache=true`
- [x] 移除每请求缓存连通性测试
  - `app/common/controller/Base.php` 中 `Cache::store(...)->set('aws_cache_test')` 逻辑迁移到后台配置保存时校验

## 二、首页后端查询优化（高优先级）
- [x] 首页小组件增加缓存（30~120秒）
  - `sidebar/announce`、`sidebar/hotTopic`、`sidebar/hotUsers`、`sidebar/hotColumn`、`common/links`
- [x] 修复 `PostRelation::processPostList` 的 N+1 查询
  - 批量获取：是否已回答、投票状态、关注状态、回答用户
  - 用内存映射替代循环内重复 `db()->where()->value()/column()`
- [x] 将在线状态清理逻辑从每请求重操作降频
  - `CheckHelper::checkOnline()` 中的过期删除改为节流执行，减少高频 `delete`
- [x] 首页游客短缓存（整页+列表组件，60秒）
  - `app/frontend/Index.php`：游客首页整页缓存，响应头返回 `X-Page-Cache: MISS/HIT`
  - `app/widget/Common.php`：首页内容列表组件短缓存，降低列表查询压力
- [x] 首页 SQL P95 采样日志
  - `app/common/controller/Frontend.php`：仅首页记录 SQL 次数、SQL 总耗时、`sql_p95_ms`、慢 SQL Top3
  - 日志文件：`runtime/log/perf_sql.log`
- [x] 基于 SQL P95 的慢查询收敛（第二轮）
  - `app/common/library/helper/UserAuthHelper.php`：导航菜单树缓存 300 秒，减少每请求 `menu_rule` 查询与树构建
  - `app/model/PostRelation.php`：默认关闭首页列表 `answer_users` 高开销聚合查询（开关：`list_answer_users_enable=Y`）

## 三、SQL与索引优化（中优先级）
- [x] 减少首页关键路径 `ORDER BY RAND()` 使用
  - 替换为随机种子/随机ID段/预计算推荐池
- [x] 为高频查询补充索引（已在生产库执行）
  - `kn_post_relation`: `idx_status_item_update(status,item_type,update_time)` / `idx_status_category_update(status,category_id,update_time)` / `idx_status_popular(status,popular_value)`
  - `kn_answer`: `idx_uid_question_status(uid,question_id,status)` / `idx_question_status_agree(question_id,status,agree_count)`
  - `kn_users_follow`: `idx_fans_friend_status(fans_uid,friend_uid,status)`
  - `kn_topic_relation`: `idx_topic_status_item(topic_id,status,item_type,item_id)`
  - 已完成 `SHOW INDEX` 复核

## 四、前端首屏优化（高优先级）
- [x] 非关键JS按需加载（第一阶段）
  - `webuploader.js` 已改为按页面加载
  - `captcha.css/captcha.js` 已改为按页面加载
- [x] 全局脚本 defer 化（第一阶段）
  - `jquery/jquery-ui/bootstrap/swiper/layui/module/tools/aws/app` 已添加 `defer`
  - 内联脚本改为 DOMReady 后执行，避免依赖未加载
- [x] 首页仅加载必要资源（第一阶段）
  - 不在首页使用的库从全局模板剥离（已完成 webuploader/captcha）
- [x] 继续压缩与缓存策略（第一阶段）
  - 已启用 Apache gzip（`mod_deflate`）规则
  - 已启用 Apache brotli（`mod_brotli`）规则
  - 已配置静态资源缓存头与 Expires 规则
  - Nginx/brotli 规则待服务器侧同步
- [x] 首页头图优化（第二阶段）
  - 新增 `top-img.webp` 并 WebP 优先加载，PNG 回退
  - `top-img.png` 已进一步压缩（约 `727KB -> 340KB`）
- [x] 首屏 CSS 阻塞优化（第二阶段）
  - 非关键 CSS 改为 `preload + onload` 延后加载（swiper/layui/highlight/module/captcha）
  - 关键 CSS 保留阻塞加载（bootstrap/fonts/app）
- [x] 公共资源按需拆分（第二阶段）
  - `highlight` 样式按控制器按需加载，首页不再下发
- [x] 字体加载优化（第二阶段）
  - `public/static/common/fonts/fonts.min.css` 增加 `font-display:swap`
  - 模板预加载 `fontawesome-webfont.woff2` 与 `iconfont.woff2`

## 六、搜索引擎索引优化（高优先级）
- [x] Sitemap 动态化与定时更新
  - 生成 `public/sitemap.xml`（首页、列表页、问题、文章、话题、用户）
  - 每日自动刷新，避免长期静态旧链接
  - 前台请求自动检查过期并后台刷新，无需手动维护
- [x] robots.txt 规范化
  - 增加 `Sitemap` 声明

- [x] 索引质量优化（第一阶段）

- [x] 索引监控（第一阶段）
