# 发布测试规则模板

用于后续 agent 按固定流程测试 FAQ 发布、FAQ 自答、带图内容发布与话题绑定，以及“直接发布”和“进入待审”两类结果。

## 1. 认证规则

- 接口优先使用请求头 `ApiToken`。
- 若 `ApiToken` 不生效，再尝试 `AccessToken`。
- 不要优先使用 `UserToken`。
- 所有发布请求都默认带上登录态相关 token，不要在正文里暴露真实 token 值。

## 2. 热门 FAQ 发现

- 先调用问题热榜接口：`/api/Question/index?sort=hot&page=1&page_size=10`
- 取最近热门问题作为 FAQ 选题参考。
- 优先选择：
  - 问题足够具体
  - 答案可以直接复用
  - 话题可以清晰分类

## 3. FAQ 发布流程

### 3.1 选题

- 围绕当前热门问题生成一个可回答的问题标题。
- 问题标题必须是明确提问句。
- 问题内容要补充背景、判断维度、待确认点。

### 3.2 发布问题

- 接口：`POST /api/Question/publish`
- 必填字段：
  - `title`
  - `detail`
  - `category_id`
  - `topics`
- 推荐字段：
  - `question_type=normal`
  - `is_anonymous=0`

### 3.3 自答

- 接口：`POST /api/Question/save_answer`
- 先写结论，再写简短解释。
- 回答结构建议：
  - 直接结论
  - 判断维度
  - 适用边界
  - 下一步建议

## 4. 带图内容发布流程

### 4.1 主题选择

- 主题应适合文章表达，不要写成纯问题句。
- 优先选择可以配图说明的主题，例如：
  - 大数据展示平台选型
  - 实时大屏能力对比
  - 数据看板与可视化平台差异

### 4.2 图片规则

- 图片必须先生成或准备好，再发布文章。
- 封面图建议使用横版比例，优先 `1200x800`。
- 图片内容要和主题一致，避免空白图、纯文字图和无关抽象图。
- 图片文件应可被网站静态路径访问。

### 4.3 发布文章

- 接口：`POST /api/Article/publish`
- 必填字段：
  - `title`
  - `message`
  - `category_id`
  - `topics`
- 推荐字段：
  - `article_type=faq`
  - `cover=/storage/generated/<file>.png`

## 5. 话题绑定规则

- 话题使用数组形式提交，尽量传 `topics[][id]`。
- 如果接口兼容字符串形式，也要优先用对象数组。
- 话题数量不要超过站点限制。
- 话题名称要和内容主题一致，避免跨领域乱挂话题。
- 当 `Topic/create` 返回 `status=pending_review` 时，说明话题已进入审核队列，此时不能再把该话题继续用于后续内容绑定。

## 6. 审核链路规则

- 如果返回文案包含“请等待管理员审核”，默认视为链路正常，不要按失败处理。
- 使用后台 `ApiToken` 做文章发文测试时，即使 token 绑定管理员账号，也应进入待审，不应直接公开发布。
- 文章接口进入待审时，优先读取：
  - `data.status=pending_review`
  - `data.approval_id`
- 话题接口进入待审时，优先读取：
  - `data.status=pending_review`
  - `data.approval_id`
- 进入待审后，记录：
  - 原始请求标题
  - 审核单 ID
  - 返回消息
  - 是否还能继续执行下游发布
- 若当前链路依赖已审核话题 ID，则必须改用已有话题，不能拿待审话题继续发文。

## 7. 验收检查

- 新发布的 FAQ 能在问题列表查到。
- FAQ 的回答能在问题详情中查到。
- 新发布的文章能在文章列表查到。
- 文章封面能正常显示。
- 文章和问题的话题都能在详情页查到。
- 热榜列表能看到新内容有基本曝光或关联痕迹。
- 如果本轮测试目标是“审核链路”，则验收改为：
  - 后台审核列表能看到新增待审话题
  - 后台审核列表能看到新增待审文章
  - 测试脚本输出 `article_status=pending_review`，且包含 `article_approval_id`
  - 返回结果里已记录 `approval_id` 或至少明确返回“等待管理员审核”

## 8. Agent 执行模板

```text
任务目标：
1. 先查热门 FAQ。
2. 发布一个新的 FAQ 问题。
3. 为该 FAQ 立即发布一条自答。
4. 再发布一篇带封面的文章。
5. 最后核对列表、详情、话题和封面是否正常。

约束：
- 认证头优先 ApiToken。
- FAQ 使用 question + answer 流程。
- 文章必须带 topic 和 cover。
- 封面图必须符合横版比例。
- 结果需要记录新建内容 ID、标题、分类、话题、封面路径。
- 如果命中审核链路，还要记录待审状态和 `approval_id`。
```

## 9. 脚本入口

- 推荐直接执行：`API_TOKEN=... python3 scripts/publish_chain.py --theme "大数据展示平台"`
- 默认会执行完整链路：
  - 发现热榜
  - 创建或复用话题
  - 发布 FAQ
  - 发布自答
  - 发布带图文章
  - 校验详情页结果
- 封面默认使用：`/static/generated/big-data-platform-cover.png`
