# Frelink 改革落地路线

本文档用于把 Frelink 从“传统问答/文章站”推进为“公开、开放、可检索、支持 AI 自动化运营的知识站”。

## 核心目标

1. 把 `www.frelink.top` 做成移动端优先的公开知识检索站。
2. 把 GitHub 项目做成可部署、可二开、可接 agent 的开源知识系统。
3. 让后端持续产出最近周期的运营信号，支持 agent 给站长提供选题、内链、专题和补文档建议。
4. 运营判断基于滚动窗口，不使用长期累计点击率。

## 已完成

### 站点定位与基础结构

- README 已改为“公开、开放、可检索的知识站”方向。
- 默认首页、帮助页、主题文案和部分导航已向知识检索结构收敛。
- 新装实例默认导航已调整为 `首页 / 主题 / 问题 / 文章 / 专题 / 帮助中心`。

### API 与运营数据层

- 新增 `Insight` 模块：
  - `POST /api/Insight/track`
  - `GET /api/Insight/summary`
  - `GET /api/Insight/keywords`
  - `GET /api/Insight/opportunities`
  - `GET /api/Insight/content_trends`
  - `GET /api/Insight/topic_trends`
  - `GET /api/Insight/recommendations`
- 新增 `analytics_event` 表，记录匿名和登录用户的曝光、点击、详情阅读。
- 桌面端和移动端列表页、详情页已接入最小埋点。

### 后台可视化

- 后台首页已加入“运营洞察”标签页。
- 支持按最近 `1 / 3 / 7 / 30` 天切换查看：
  - 搜索次数
  - 曝光次数
  - 点击次数
  - 窗口 CTR
  - 搜索缺口
  - 内容热点
  - 主题热点
  - Agent 建议
- 后台已加入 `Agent 执行简报` 文本区，可直接复制给 agent。
- 后台已新增简报导出能力：
  - `GET /admin.php/index/insightBrief?days=7&format=markdown`
  - `GET /admin.php/index/insightBrief?days=7&format=json`

## 当前信息架构建议

前台一级栏目建议固定为：

- 首页
- 主题
- 问题
- 文章
- 专题
- 帮助中心
- 关于

不建议当前阶段作为一级核心入口：

- 想法
- 大咖
- 专栏
- 创作者

说明：

- `主题` 做 SEO 聚合和内链中枢，应前置。
- `问题` 承接搜索流量和 FAQ。
- `文章` 承接教程、研究综述、思想碎片整理稿。
- `专题` 做系列内容和热点收口。
- `帮助中心` 承接术语、规则、文档、安装和接入说明。

## 改革推进顺序

### 第一阶段：数据基础打通

目标：先让系统知道“用户在搜什么、点什么、读什么”。

执行项：

1. 执行 `docs/analytics-upgrade.sql`
2. 执行 `docs/nav-upgrade.sql`
3. 确认 `analytics_event` 有新数据写入
4. 确认后台首页“运营洞察”可正常打开
5. 确认 `Insight` API 可返回最近窗口数据

验收标准：

- 最近 24 小时有曝光、点击、详情阅读写入
- `summary`、`opportunities`、`content_trends` 有有效结果
- 后台能切换 `1/3/7/30` 天窗口

### 第二阶段：移动端体验重做

目标：让用户在手机上更快完成“搜索 -> 阅读 -> 下一步探索”。

当前进展：

- 已补充问题/文章详情页的 `30 秒看懂` 区域。
- 已补充问题/文章详情页的 `下一步阅读` 区域。
- 已新增用于摘要点和下一步阅读的公共 helper。
- 已把移动端首页改为搜索优先，并新增：
  - `最近有人在搜`
  - `本周值得看`
  - 次级入口折叠区
- 已修复线上前台 rewrite，详情直链重新可用：
  - `/article/{id}.html`
  - `/question/{id}.html`
- 已开启线上 `mobile_enable`，手机 UA 访问已命中移动模板。
- 已新增显式移动入口：
  - `/m/`
  - `/m/question/{id}.html`
  - `/m/article/{id}.html`
- 已把线上首页主题文案切换为“公开、开放、可检索的知识站”。
- 已清理移动端入口排障期间的临时探针代码和运行日志。

执行项：

1. 首页改为搜索优先，而不是传统社区流优先
2. 问题/文章详情页顶部加入“30 秒看懂”区域
3. 在详情页加入“下一步阅读”而不是简单相关文章
4. 减少移动端底部导航里与知识消费无关的入口
5. 优化长文阅读间距、字号和折叠逻辑

验收标准：

- 首页首屏优先出现搜索和核心入口
- 详情页阅读路径更短
- 移动端单页跳出率和详情阅读量提升

### 第三阶段：Agent 辅助运营

目标：让 agent 能读懂站点近况，但保留站长审批。

执行项：

1. 用 `Insight` 接口和 CLI 报告给 agent 提供输入
2. 让 agent 每日输出：
   - 搜索缺口
   - 低曝光高 CTR 内容
   - 应扩展主题
   - 帮助文档缺口
3. 站长审核后再发文或建专题
4. 在问题/文章发布页直接显示最近窗口的搜索缺口、推荐标题和推荐主题
5. 提供 `publish_assist` 接口给 agent 和后台使用

验收标准：

- 每天能生成一份最近 7 天运营报告
- 站长能按建议补问题、文章、帮助和专题
- 不允许 agent 直接删内容或直接正式发布
- 发布页能看到最近 7 天的内容方向提示

### 第四阶段：发布辅助与专题增长

目标：把运营洞察转成内容生产效率。

执行项：

1. 发布页加入标题、摘要、话题、相关主题建议
2. 根据搜索词缺口生成选题池
3. 把高热搜索词聚合成专题和帮助章节
4. 做站内推荐位和主题页强化曝光

验收标准：

- 内容新增不再靠纯人工拍脑袋
- 站内热点和搜索词能持续转化成内容资产

## 服务器执行清单

拿到 SSH 后按下面顺序执行：

1. 备份数据库
2. 备份线上当前代码目录
3. 上传或拉取最新代码
4. 执行数据库更新：
   - `docs/nav-upgrade.sql`
   - `docs/analytics-upgrade.sql`
5. 清缓存
6. 执行 sitemap 重建
7. 访问后台首页确认“运营洞察”页签
8. 访问以下接口做 smoke test：
   - `/api/Common/config`
   - `/api/Insight/summary?days=7`
   - `/api/Insight/opportunities?days=7&limit=5`
9. 访问前台首页、问题页、文章页、详情页，确认埋点开始写入

## 运维与 Agent 接入建议

### 可直接开放给 agent 的能力

- 读取 `Insight` 相关只读接口
- 调用 `insight:report` 生成日报
- 读取搜索缺口和热点内容
- 生成选题建议、帮助文档建议、专题建议

### 不应直接开放给 agent 的能力

- 删除内容
- 改权限
- 正式发布线上文章
- 直接操作生产数据库结构

### 推荐命令

```bash
php think insight:report --days=7 --limit=5 --format=markdown
php think insight:report --days=7 --limit=10 --format=json
php think sitemap:build --base-url=https://www.frelink.top
```

## 下一步开发重点

优先级从高到低：

1. 移动端搜索优先首页
2. 详情页“30 秒看懂”摘要区
3. 发布页选题建议与主题建议
4. 每日报告和定时任务
5. Topic Graph 数据层预埋

## 说明

本轮改革不是一次性做完所有 AI 功能，而是先把“运营数据 -> 后台洞察 -> agent 建议 -> 站长决策”这条链路打通。
