## 2026-03-14（已在生产执行）
- 目的：补齐高频查询索引，降低首页和列表页慢查询风险。

```sql
-- post_relation 列表页核心过滤/排序
ALTER TABLE `kn_post_relation`
ADD INDEX `idx_status_item_update` (`status`,`item_type`,`update_time`),
ADD INDEX `idx_status_category_update` (`status`,`category_id`,`update_time`),
ADD INDEX `idx_status_popular` (`status`,`popular_value`);

-- answer 用户是否已回答/问题回答用户
ALTER TABLE `kn_answer`
ADD INDEX `idx_uid_question_status` (`uid`,`question_id`,`status`),
ADD INDEX `idx_question_status_agree` (`question_id`,`status`,`agree_count`);

-- users_follow 关注关系查询
ALTER TABLE `kn_users_follow`
ADD INDEX `idx_fans_friend_status` (`fans_uid`,`friend_uid`,`status`);

-- topic_relation 话题聚合查询
ALTER TABLE `kn_topic_relation`
ADD INDEX `idx_topic_status_item` (`topic_id`,`status`,`item_type`,`item_id`);
```

- 执行结果：7 个索引全部创建成功，并已通过 `SHOW INDEX` 复核。

## 说明
- 如后续版本涉及数据库调整，请按以下格式追加：

```sql
-- YYYY-MM-DD 变更说明
ALTER TABLE your_table ADD COLUMN example_col VARCHAR(64) NOT NULL DEFAULT '' COMMENT '示例字段';
CREATE INDEX idx_example_col ON your_table(example_col);
```

## 2026-03-14（管理端 P2，待在生产执行）
- 目的：优化后台高频列表（问题/文章/话题/用户）的过滤与排序链路，减少 filesort 与全表扫描风险。

```sql
-- 如你的实际前缀不是 aws_，请替换为线上表前缀（例如 kn_）

ADD INDEX `idx_status_id` (`status`, `id`);

ALTER TABLE `aws_topic`
ADD INDEX `idx_is_parent_id` (`is_parent`, `id`);

ADD INDEX `idx_status_forbidden_uid` (`status`, `forbidden_ip`, `uid`),

```

- 回滚 SQL：

```sql
ALTER TABLE `aws_question` DROP INDEX `idx_status_id`;
ALTER TABLE `aws_article` DROP INDEX `idx_status_id`;
ALTER TABLE `aws_topic` DROP INDEX `idx_is_parent_id`;
ALTER TABLE `aws_users` DROP INDEX `idx_status_forbidden_uid`, DROP INDEX `idx_forbidden_uid`;
```
