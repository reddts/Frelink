# 管理端优化计划（`app/backend`）

## 范围与目标
- 页面范围：`/admin.php/*`（重点关注你提供的 `https://www.frelink.top/admin.php/explore/` 链路）
- 目标：
  - 降低后台页面首开与切换等待时间
  - 降低后台公共布局每请求重复查询
  - 不影响现有后台功能与权限逻辑

## 现状结论
- `admin.php/explore` 当前未登录时会跳转到 `/account/login`，属于鉴权行为。
- 后台公共基类每次请求都会执行：
  - 菜单树构建（权限过滤 + 树结构拼接）
  - 通知统计（多张表计数）
- 后台首页统计页面（`app/backend/Index.php@index`）包含多次实时 `count()`。

## 优化项（P1）
- [x] 后台菜单树短缓存（按用户 + 路由）
  - 位置：`app/common/library/helper/AuthHelper.php`
  - 目标：减少菜单查询与树构建开销
- [x] 后台通知统计短缓存（30s）
  - 位置：`app/common/library/helper/NotifyHelper.php`
  - 目标：避免每请求多次 `count` 重复执行
- [x] 后台首页统计短缓存（60s）
  - 位置：`app/backend/Index.php`
  - 目标：降低首页并发下的数据库压力
- [x] 后台局部请求关闭通知聚合查询
  - 位置：`app/common/controller/Backend.php`
  - 目标：减少 `_ajax/_pjax` 子请求的重复计数开销

## 优化项（P2）
- [x] 后台公共资源按页面分组加载（避免全量库一次性加载）
  - 位置：`app/backend/view/block.php`
- [x] 后台列表页查询链路复核（排序/筛选字段索引匹配）
  - 位置：`app/backend/content/*`, `app/backend/member/*`
  - 已完成：
    - 列表排序字段白名单与排序方向规范化（`Question/Article/Topic/Users`）
    - 用户列表筛选条件合并修复（避免覆盖搜索条件）
    - 用户名模糊搜索修复（`LIKE '%keyword%'`）
    - 新增管理端高频列表索引 SQL（见 `数据库修改.md`，待生产执行）

## 验证方式
- 功能验证：
  - 后台登录、菜单展开、通知入口、首页统计展示正常
- 性能验证：
  - 对比优化前后后台首页 `TTFB`
  - 对比后台首页与常用列表页 SQL 次数与耗时（抽样）
