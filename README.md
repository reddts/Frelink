# Frelink

Frelink 是一个面向个人站长、垂直社区和团队知识库的开源知识检索与问答系统。

它的目标不是制造付费门槛、诱导关注或内容割裂，而是把问题、文章、主题、专题和帮助文档组织成一个可以持续检索、持续沉淀、持续复用的开放知识站。

## 产品定位

- 公开知识检索站：让访客优先看到完整内容，而不是登录墙和付费提示。
- 垂直主题社区：用主题聚合问题、文章、专题和帮助文档，形成稳定的知识结构。
- 可二次开发的开源程序：提供 Web、移动端和 API 基础能力，适合继续扩展自动化和 AI 能力。

## 适用场景

- 个人公开知识站
- 行业垂直问答社区
- 企业内部知识库
- 开源项目文档与 FAQ 中心
- 围绕主题聚合内容的搜索型站点

## 安装方式

### 1. 环境要求
- PHP >= 7.4
- MySQL 5.7+
- Nginx/Apache（站点根目录指向 `public/`）
- PHP 扩展：`json`、`iconv`、`zip`、`curl`、`gd`、`zlib`、`openssl`

### 2. 获取代码与依赖
```bash
git clone https://github.com/reddts/Frelink.git
cd Frelink
composer install
```

### 3. 配置数据库
1. 创建数据库（建议 `utf8mb4`）。
2. 导入安装 SQL：`install/sql/install.sql`。
3. 配置数据库连接（按你的部署环境修改配置文件/环境变量）。

### 4. 安装锁与目录权限
- 安装完成后创建安装锁文件：`install/lock/install.lock`（避免重复进入安装流程）。
- 确保以下目录可写：
  - `runtime/`
  - `public/storage/`
  - `public/backup/`（如启用备份功能）

### 5. Web 服务器配置
- 站点入口：`public/index.php`
- 后台入口：`public/admin.php`
- 伪静态可参考：`public/nginx.htaccess`、项目根目录 `nginx.htaccess`

## 核心能力
- 知识问答：提问、回答、评论、收藏、关注
- 开放检索：问题、文章、主题、帮助文档统一搜索
- 主题聚合：相关主题、主题合并、无限级主题
- 内容组织：文章、专题、帮助、页面等多种内容形态
- 社区治理：审核记录、敏感词、内容审核流程
- 用户体系：用户组、权限、积分、威望、认证
- 多端支持：网页端、移动端、API
- 扩展机制：插件系统、钩子、定时任务

## 推荐的信息架构

- 首页：搜索、推荐流、公告、核心主题
- 问题：承接长尾搜索词和讨论入口
- 文章：承接教程、案例、方案总结
- 主题：作为聚合页连接问题、文章、专题
- 专题：做系列化内容整理
- 帮助：做术语、规则、文档和新手指南

## 对外表达建议

- GitHub 仓库强调“开源知识检索与问答系统”
- 演示站强调“公开、开放、可检索的知识站”
- 尽量避免把“企业社区、行业社区、知识付费、问答系统”同时堆在首屏
- 优先突出开放阅读、主题聚合、搜索沉淀、API 可接入这四个差异点

## API 与自动化

- API 入口控制器位于 `app/api/v1`
- 当前仓库已补充基础 API 接入说明，见 `docs/api-v1.md`
- 建议把 agent 的第一阶段能力放在：
  - 内容巡检
  - 搜索词选题
  - 站点收录检查
  - 发布后 smoke test
  - 运维告警汇总
